You are an expert quantitative strategy developer. Given a market summary for a stock, design and implement a quantitative trading strategy.

RULES (strictly follow ALL):
1. Output exactly ONE Python code block (```python ... ```) containing a single Strategy class.
2. The class MUST subclass `Strategy`.
3. All indicators MUST be wired through `self.I(FUNC, ...)` in `init()`.
4. In `next()`, only reference current/past data with negative indices: `[-1]`, `[-2]`. NEVER use positive indices.
5. Include `self.buy()` and `self.sell()` calls in `next()`.
6. Define tunable params as class-level attributes (e.g. `n_fast = 20`).
7. Add risk management: use `self.buy(sl=...)` for stop-loss OR explicit exit rules.
8. Do NOT define your own helper methods on the class for indicators — use the TA functions listed below directly.

AVAILABLE NAMES (already in scope — do NOT import anything):
- `Strategy` — base class
- `np`, `pd` — numpy and pandas
- `crossover(a, b)`, `cross(a, b)`, `barssince(cond)` — from backtesting.lib

TA HELPER FUNCTIONS (use these directly with self.I — do NOT reimplement them):
| Function                              | Returns                          |
|---------------------------------------|----------------------------------|
| `SMA(values, n)`                      | Simple Moving Average            |
| `EMA(values, n)`                      | Exponential Moving Average       |
| `RSI(values, n=14)`                   | Relative Strength Index (0–100)  |
| `MACD(values, fast=12, slow=26)`      | MACD line                        |
| `MACD_SIGNAL(values, 12, 26, 9)`      | MACD signal line                 |
| `BBANDS_UPPER(values, n=20, k=2.0)`   | Bollinger upper band             |
| `BBANDS_MID(values, n=20)`            | Bollinger middle band            |
| `BBANDS_LOWER(values, n=20, k=2.0)`   | Bollinger lower band             |
| `ATR(high, low, close, n=14)`         | Average True Range               |

| `HIGHEST(values, n)` / `LOWEST(values, n)` | Rolling max / min          |

FORBIDDEN — these patterns will cause runtime errors:
- `import ta`, `import talib`, `import pandas_ta`, `import finta`, `import stockstats`
- Calling `ta.*`, `talib.*`, etc.
- Defining indicators as instance methods (e.g. `self.SMA(...)`) — call the module-level helpers directly.
- `np.convolve(values, weights, mode='valid')` for a moving average — this returns a SHORTER array and breaks the `self.I()` length contract. **Use `self.I(SMA, close, n)` instead.**
- Any function passed to `self.I()` that returns an array shorter than the input. Length MUST be preserved (rolling/ewm-based math is fine; convolve with mode='valid' is not).
- Using `mode='valid'` anywhere — it truncates output.

CORRECT EXAMPLE:
```python
class SmaRsiStrategy(Strategy):
    fast = 20
    slow = 50
    rsi_period = 14
    rsi_oversold = 35
    rsi_overbought = 65

    def init(self):
        close = self.data.Close
        self.sma_fast = self.I(SMA, close, self.fast)
        self.sma_slow = self.I(SMA, close, self.slow)
        self.rsi = self.I(RSI, close, self.rsi_period)
        self.atr = self.I(ATR, self.data.High, self.data.Low, self.data.Close, 14)

    def next(self):
        if crossover(self.sma_fast, self.sma_slow) and self.rsi[-1] < self.rsi_overbought:
            sl = self.data.Close[-1] - 2 * self.atr[-1]
            self.buy(sl=sl)
        elif crossover(self.sma_slow, self.sma_fast) or self.rsi[-1] > self.rsi_overbought:
            self.sell()
```
